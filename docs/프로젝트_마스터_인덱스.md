> **상태:** 5축 + Custom SoC 오버레이 + 부품 첫 축(서버) **전부 1바퀴 graduate 완료.**

## 축 스키마 (최종 = 2 응용처축 + 2 종속층 + 1 오버레이)
```
[응용처축]   Mobile AP            HPC·DC
   ══ Custom SoC 오버레이 (두 축 가로지름, 응용처 ~50% 종속·이중속성, captive 태그로 축별 구현) ══
[종속층]     Foundry ──────── Packaging
                                 ├ 기술 tier      : 2.5D / 3D / FOPLP-PLP
                                 ├ Package Module tier [신설] : CPO · Si Cap · BSPDN · CIS PKG
                                 └ 부품 2-tier     : ABF·유리기판 / MLCC / 인덕터
                             [배경] legacy = 병목 무관
```
**구조적 발견:** make-vs-buy 오버레이가 종속층 3곳(Custom SoC → Packaging → Package Module)을 관통.

## graduate된 벼리 — 켜뮤 목록

### HPC·DC
- **공동병목 = 다각형(≥4 꼭짓점).** interposer · HBM bit · buffer die · leading-edge logic pool이 lockstep(5~6 확장가능, spider chart). buffer die 세대상승이 **A14(2029E)에서 logic과 수렴.** (CoPoW→CoPoS 명명 통합)
- **NVIDIA 해자 무게중심 이동.** CUDA(SW) → 공급망 배정권(CoWoS 우선·HBM4 선점). → Package 귀속. 반론: 배정권 희석 가능.
- **수직통합 ≠ 강점.** top-two 아니면 이음새 침투(Samsung HBM3E 실증). 반론: 턴키 리드타임 값어치(AMD MI400 경계).
- **[이송]** 출하량 vs 매출 착시 → 예측 섹터. 헤드라인-실질 분리는 체크리스트 한 항목.

### Custom SoC (오버레이)
- **A — 사업모델 오버레이.** 응용처 ~50% 종속·이중속성, turn-key HW/SW(ref Broadcom·Socionext). pkg 설계까지면 'Custom Biz' 분리.
- **B — 경쟁축 = 발주자별 make-vs-buy 궤적.** 진짜 경쟁자 = in-house(Apple=make 극단). make/buy는 왕복.

### Mobile AP
- **경계 = merchant AP만**(Qualcomm·MediaTek). captive(Apple A·Tensor)는 오버레이로 이월.
- **A — captive 태그 = 오버레이 축별 구현체.** `#captive→[발주자]`. captive≠custom → confidence 속성 필수.
- **[Foundry park]** 공급단 원가(웨이퍼 가격 등) = 공급자 축 attribution.

### Foundry
- **Watch 등급 트리아지.** 10년내 매출 난망 격리 + 재소환 트리거 1개 필수. 편입: Terafab(PDK0.9)·후면전력(양산라인)·Intel14A(외부고객).
- **판단:** Samsung SF2 ~55%(실효 ~40%), 70% 보도 배제. PMIC·CoWoS → Packaging.

### Packaging
- **A — Package Module tier 신설.** CPO·Si Cap·BSPDN·CIS PKG. 부품층도 순수 기법도 아닌 '사이 계층'(하위 부품군 통합 + eco-chain 이질).
- **B — make-vs-buy 오버레이 종속층 3층 관통.** 방향 반전: PKG선 파운드리가 CoWoS 배분권으로 buy측(OSAT) 통제. Package Module엔 "본업이냐(업의 본질)"가 축. 반론: 본업 ≠ top-2 (본업성 × top-2 2×2).
- **C — CPO eco-chain 이질 = 이음새 침투 실증.** TSMC SiPh 약체(COUPE 이제 진입) vs NVIDIA Lumentum·Coherent 직투('26.3) = 발주자가 파운드리 non-top-2 이음새 직접 메움. → HPC·DC 수직통합 판단에 링크.

### 부품 (삼성전기 서버축)
- **A1 — SAM 경계** = 현 사업범위 유지(명시적 벤처뉴스 전까지).
- **A2 — content** = 수량 × 혼합 고정단가, 물량만 변수.
- **A3 — 강도** = 총시장 성장 아닌 현 점유율 전망으로 rebase.
- baseline: `srv-f01~05`, bom_implication(MLCC·substrate·inductor·module 전부 up). 신사업 브릿지: srv-f01→Si Cap, srv-f02→유리기판.

## 메타원칙 (graduate됨)
- **component attribution = allocation-ratio 문제**(배타 정렬 아님). ratio 출처 신뢰등급(disclosed/estimated/back-calculated) 필수.
- **페르소나 스파링** = 질문 scaffolding만. 도메인 확답은 실제 전문가/삼성전기 contact 필요.

## 남은 백로그 (섹터 아님 · dangling)
1. **명명 통합** CoPoS / CoPoW / CoWoP / FOPLP (HPC·DC 핀 + PKG dangling 합쳐 1회 정리)
2. **CoWoS 배분 벼리** 재사용 vs 신규 — 이미 승격된 "배분권" 판단과 중복 판정
3. **BSPDN 3사 접근차** 귀속 판정 (Foundry ↔ Package Module tier 겹침)
4. **부품 2-tier 실측 수치**(MLCC·인덕터·기판) + `srv-f05` 서버 대수 트렌드 → 삼성전기 contact / 검색 1회
5. **Silicon Cap datapoint** 미확보
6. **AI PC/Notebook 축 편입 판정** — PKG 소화 완료됐으니 이제 리트머스(고유 demand vertex 유무) 답 가능
7. **부품 baseline이 서버축만** = 타 응용처(Mobile 등) 부품 파생 미착수
